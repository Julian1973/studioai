# Crystal Bears Studio — operating instructions for Claude Code

Source of truth order: CRYSTAL_BEARS_LOCKED_CANON.md (what the show is) > CRYSTAL_BEARS_STUDIO_BIBLE.md (how it is made) > this file. Where they touch, canon wins.

Every session: read this file, TICKET_PACK_001.md and the current phase of WORLD_CLASS_ROADMAP.md before writing any code. Work tickets in the pack's order of play, one commit per ticket, the commit message naming the ticket. A ticket is done only when its Definition of Done is true AND the Bible agrees — never merely when the code compiles.

Hard rules (never work around these):
1. Gates are hard locks. Never advance past an unsigned gate; never sign past an open BLOCK.
2. Taste lives in prose (the chair skills); law lives in code; canon lives in data. Anything found in the wrong layer moves the day it is found.
3. No output is ever hand patched. A fault routes through a retake with a layer diagnosis (keyframe / brief / reference / take).
4. Law 5: the voice lives in the render. One combined @Audio1, supplied by the Voice Director and lip-synced by Seedance; no native-voice fallback (a beat with dialogue whose V3 track fails REFUSES to render); no post voice swap. cb_post has no swap function by design — do not add one.
5. Never describe a character's appearance in prompt text. Identity comes only from the reference images. Binding a name to its reference slot in the prompt text ("Fuzzby is the bee from @图2") is allowed and is now the standard (Julian, 2026-07-05, the blessed v4 template — reversing this rule's earlier "names live only in the audio" wording); what stays absolutely forbidden is DESCRIBING the character — no physical/appearance text anywhere, ever. The distinction: a name is a label, not a description.
6. Canon is edited ONLY at root CRYSTAL_BEARS_LOCKED_CANON.md, then `python3 tools/sync_canon.py`. The skill copies are generated; editing one is drift. `--check` must pass before any sign off.
7. When code becomes sharper than the documents, the documents update in the same commit.
8. Restructure work follows RESTRUCTURE_SPEC_T30.md: one phase per commit, baseline byte-identical prompts proven after each phase.
9. A temporary state (e.g. a pollen moustache) resolves WITHIN the take it started in — it never carries across a take boundary (T2 ruling, 2026-07-02, Julian). There is no continuity-tail/previous-clip-tail chaining mechanism; do not re-add one without a fresh ruling.

ACTION-STATE FIDELITY (2026-07-03, Julian — "if the first scene is them flying through the meadow, then they need to be flying through the meadow"): whenever a beat's own action text names an active locomotion verb, the KEYFRAME must show that state concretely, not just gesture at it — checked by cb_qa's new ACTION_STATE_MISMATCH (KEYFRAME_DONE.md) and generated by cb_prompts.build_keyframe_prompt's WINGS/FLIGHT-ENERGY law (asymmetric mid-downstroke wings, forward-leaning body, legs tucked — never both wings spread symmetrically with the body hanging vertical like a puppet at rest, which reads as floating in place and forces Seedance to invent a standstill-to-motion transition instead of continuing existing motion). Generation and QA ask for/verify the SAME concrete criteria, not a vague "does this look dynamic" taste call — a generic ask is what let the first real keyframes pass QA while still reading static. Deliberately excludes "hover" — the default bee state, not inherently wrong.
17. VISION-QA CHECKS MUST STATE CONCRETE VISUAL CRITERIA, NEVER SUBJECTIVE QUESTIONS (Julian's Ruling, 2026-07-03). Worked example: ACTION_STATE_MISMATCH's first draft asked the vision model a vague question — "does this pose read static, posed, standing still, or passively floating?" — and the model PASSED both of the known-bad 1.B1/1.B2 keyframes clean; it is a subjective taste call a generous grader waves through. Rewritten to ask two concrete, checkable things instead — are the wings SYMMETRICAL (fail) or ASYMMETRIC/mid-downstroke (pass)? Is the body VERTICAL with legs dangling (fail) or LEANING FORWARD into the direction of travel (pass)? — and it correctly failed both. Every vision-QA check added from here must name the specific, literally-checkable visual feature(s) that separate a pass from a fail (an angle, a count, a symmetry, a position) — never phrase the ask as an open judgment call ("does this look right/dynamic/off").

    THE FIDELITY LAW (Julian's ruling, 2026-07-06, CLEAN RESTART ON v5 — written in as the governing principle above every other law this rule states, not an addition alongside them): nothing is invented that canon already defines. Everything on screen is pulled from the bible — in the bible's own words — or it does not render.
    - **Character contract**: every character's acting DNA is extracted VERBATIM from their bible entry — movement, rhythm, comedy register, contrast pairings — locked in the character data, quoted into every prompt they appear in. No paraphrase, no softening, no session ever rewrites it. The bible is the only editor. Mechanically: `cb_segprompt._v5_acting_dna_source` quotes `actingNote` (Fuzzby/Zenny — pure movement/comedy, appearance-free by construction) or falls back to `bible.mannerisms` (the 9 bears, who have no actingNote) — the fallback is a FLAGGED gap, not a silent acceptance: several bears' mannerisms text mixes in appearance/wardrobe detail that would violate rule 5 if quoted verbatim into a shipped prompt, and only Julian's own edit of the bible entry can close that gap, never a Claude Code edit at prompt-build time. `bible.dos[0]`/`bible.donts[0]` (the Always/Never lists) feed the beat's own staging clause and negative line, verbatim, index-0 deterministic — never curated for "relevance," since choosing which existing sentence is most relevant is itself a form of authoring.
    - **Scene contract**: every scene's identity — location, light, atmosphere, register — is pulled from the script and scene canon, locked as the bubble's three constants (rule 35). A scene renders as itself.
    - **Style contract**: the look-and-feel is the locked style law plus the plate, verbatim, every prompt, every beat, every scene, forever (`cb_segprompt._v5_style`, unconditional).
    - **Enforcement**: every one of these lives as a locked data field with its bible source cited (`cb_segprompt._v5_acting_dna_source` returns the field it drew from for exactly this reason). `cb_preflight.check_characters_technical` blocks any character appearing in a beat whose store has neither `actingNote` nor `bible.mannerisms`, and neither `bible.dos` nor `bible.donts`, populated — a prompt whose character content can't trace to a locked field never compiles. Drift becomes a build error, not a mood.
    - **THE ACTINGDNA RETRACTION** (same ruling, same day): an earlier v5 draft added a NEW `characters.json` field, `actingDNA`, hand-authored per character (partly from Julian's own capsule phrases, partly synthesized from bible prose). Julian's correction was explicit and immediate — "STOP any actingDNA authoring or extraction — the content already exists" — because that field was authored prose, not a verbatim quotation, and so violated this very law the moment it was written. RETRACTED the same day: all 11 entries removed from `characters.json`, restoring the file to its pre-session state; `cb_preflight`'s per-character check now requires `actingNote`/`mannerisms` + `dos`/`donts` instead. `actingRule` (a pre-existing, similarly-condensed one-line field, rule 33) is likewise retired from every check and from the emitter — v5 reads neither `actingRule` nor `actingDNA` anywhere; the character's own real store is the only source.
    - **THE WORD-BUDGET CONSEQUENCE, FLAGGED**: pulling `actingNote`/`mannerisms` verbatim plus `dos[0]`/`donts[0]` per cast member is real, richer, more-cited content than the retracted `actingDNA` field — and it costs more words. On Scene 1 (Fuzzby + Zenny, both speaking in most beats), this pushed 4 of the 5 beats over the 400-word hard cap (402-427 words); only 1.B1 (the scene's shortest cast footprint) stays clean at 394. This is not a bug to quietly work around — it is the real, honest cost of two full characters' verbatim-cited acting register appearing in every independently-generated beat of a two-character scene, and it is reported to Julian exactly as measured, not trimmed to fit.

    THE HIERARCHY: Episode → Scene → Beat.

    THE SCENE BUBBLE (rule 35): a scene's identity is its THREE LOCKED CONSTANTS — the scene plate, the ambient bed, the style law — held VERBATIM across every beat inside it. A scene boundary is a FULL, DELIBERATE RESET: new plate, new ambient bed, a fresh canon-generated anchor keyframe for the scene's own opening beat, relay depth back to zero, no visual continuity owed across the boundary. Every law below applies identically inside every bubble.

    THE BEAT: one gag arc, 15 seconds hard-locked — 13s action + 2s settle (the Handle Doctrine, rule 20). Never two arcs in one take.

    LAYER 1 — THE INVARIANT SKELETON (machine-filled, identical logic every prompt; a law here can be forgotten only if it's convention rather than code — see the audit this rule commissioned, PROMPT_LAWS_AUDIT.md):
    1. One beat, one prompt, one gag arc. 15s = 13s action + 2s settle. Never two arcs in one take.
    2. Opener vs relay vs junction type: a scene's first beat opens from its generated canon anchor keyframe. Every beat after declares its OWN junction type (a beat-level field, `junctionType`, rule 31) — `intentional_next_shot` (THE DEFAULT: beats join by CUT, not freeze — a new gag arc, a fresh camera setup, energy already in frame) or `seamless_continuation` (ONLY where the director's own cut explicitly continues one shot across the boundary) — and opens from the previous beat's harvested settle frame either way. Omission is NEVER read as seamless — that stays the rarer, explicitly-declared case.
    3. The reference stack, every relay prompt, each entry one declared job: @图1 — for a `seamless_continuation` beat, the locked opening frame (point anchor); for an `intentional_next_shot` beat (the default), a STATE REFERENCE only — identity, declared carry marks, lighting and world-position carry forward exactly; camera, pose, energy and motion do not (rules 31/33/34). @Video1 — motion energy and action continuity ONLY; never camera framing, shot size or composition, which comes from THIS beat's own direction (the corrected line, FIXED 2026-07-05 lock-in night — the earlier wording, "previous visual motion continuity and energy," still left framing implicitly copyable). Character turnarounds — identity anchor, one per character, held every beat. Scene plate — duration anchor, the environmental constant for the whole clip (lighting, palette, texture), never a framing or pose reference. @Audio1 — the sole vocal source, covering lines, hums and sing-song rhythm alike; performance anchor: timing, mouth rhythm and full physical performance, camera locked whenever any character speaks.
    4. The @图1 description is a photograph for a `seamless_continuation` beat (content, not choreography — no temporal verbs, no imperatives, no "end on") or the fixed state-carry template for an `intentional_next_shot` beat (Law 3) — never a mix of the two, and never the predecessor's full endState sentence stuffed in wholesale (rule 33's rewrite of `_v4_state_carry`).
    5. Anti-hold — `seamless_continuation` ONLY: frame one IS @图1 exactly, then motion moves decisively away, immediately; never held, repeated or returned to (rule 26). An `intentional_next_shot` beat (the default, rule 31) never opens on @图1's frame at all — it opens on its own fresh camera setup, carrying only @图1's declared state forward; the model is never asked to hold or repeat a pose that isn't this beat's own.
    6. Audio law: all vocal sound — lines, hums, exclamations, sing-song — is the V3 performance fired in as @Audio1. Seedance generates no voice-like sound of its own. The prompt never contains the spoken words — dialogue binds speaker, timing and expression only, by name and ordinal.
    7. Ambience is scene-property: one locked ambient-bed line, word-for-word identical across every prompt in the scene (the Scene Bubble Law). Per-beat SFX layers on top; ambience never appears inside any other field.
    8. Camera law: locked during any SPOKEN line (rule 38, 2026-07-06 — a continuous hum or sing-song is motion-exempt); motion only in non-speaking action; species-scaled (mechanical, from the cast — "bee-scale") and naming the SPECIFIC eyeline/action characters when the beat has declared an `opensOn` pair (rule 33), falling back to generic phrasing otherwise.
    9. The living settle: every clip ends on ITS OWN completed beat, in character — read from the beat's own authored `endState` VERBATIM (rule 33; never the old generic idle-life placeholder once `endState` is authored), so a manic character stays visibly manic at rest and a deadpan character gets its own full pause. NEVER a restatement of the previous beat's pose — the settle is this beat's OWN distinct moment (advisory-checked by `check_settle_distinctiveness`); that settle is the next beat's harvest.
    10. The standing negatives, every prompt, in full — Crystal World Rule and Wing Law always — plus the beat's own `stagingProhibited` items additive on top (rule 30), never fewer than the standing count. CORRECTED 2026-07-08 (contradiction sweep): this used to say "six," describing v3/v4-era `_v3_negatives` (species-conditional, deleted in THE DEFINITIVE BUILD purge, rule 40) — the live mechanism (`cb_segprompt._standing_negatives()`) has been an unconditional ELEVEN-item list since rules 43/44 (2026-07-06); this line just never caught up, the same staleness already found and fixed in MANIFEST.md the same day.
    11. Character law: identity by reference only, name-welded directly to its @图N slot ("Fuzzby is the bee from @图2," rule 5/30) — a name is a label, never a description; no physical/appearance text anywhere, ever.
    12. Style law and world text: verbatim from the show profile and scene data, never retyped, never paraphrased.

    LAYER 2 — THE CREATIVE SLOTS (director-authored; the ONLY things that change beat to beat):
    1. The beat's own junction type (Law 2) — defaults to cut; declare `seamless_continuation` only when the shot genuinely doesn't travel.
    2. Action prose: the beat's own gag, ERRATIC IN CHARACTER, PRECISE IN CHOREOGRAPHY (rule 33) — every manic action a SPECIFIC NAMED GAG with cause and consequence (rockets, brakes too late, loops once, stops); adjective-chaos ("wildly," "crazily," a generic frenzy word with no physical beat behind it) is BANNED as unreadable — the model executes actions and butchers adjectives.
    3. `opensOn: {who, action}` — WHO the camera opens on and their immediate mid-motion state; the Coverage Law's reaction-bridge mechanism, rendered as a fixed "OPEN ON {NAME} mid-motion: {action}." sentence — a foil character's look or turn (Zenny's head-turn) is the house bridge whenever the cut travels.
    4. `carryMarks` — a SHORT phrase (never a full sentence) naming what specifically, visibly persists (a moustache, leg pollen) — the ONLY thing the join-check's STATE tier hard-gates on (rule 36), and what the NEXT beat's own @图1 clause quotes.
    5. `endState` — directing prose for THIS beat's own ending, written as its own distinct moment in character, never a restatement of the predecessor's pose. `endStateStill` — the same moment as a static photograph, hand-authored in parallel (rule 27), never auto-derived from `endState`.
    6. `actingRule` (per character, a new one-line `characters.json` bible field) + `actingContrast` (per beat, which characters play off each other and how) — feed the `acting_rules` field.
    7. Dialogue bindings (who speaks, expression direction), tone, per-beat foreground SFX, `stagingProhibited` (beat-specific failure modes, merged with the standing negatives — eleven as of rules 43/44, see Law 10's 2026-07-08 correction above).

    THE COVERAGE LAW (rule 34): a scene is ONE CONTINUOUS SPATIAL BUBBLE. Every beat opens on a NEW ANGLE WITHIN the same space, close to where the previous beat ended, motivated by eyeline or motion (Layer 2.3's `opensOn`) — never a relocation, never a fresh establishing wide mid-scene (the wide belongs to the scene's opening beat only). The join-check's STATE tier gains COVERAGE as a fourth, always-on criterion alongside STATE/LIGHT/GEOGRAPHY: does the opening read as a continuation within the same space, or a re-establish? Distinct from GEOGRAPHY (which only asks "is this the same world at all") — a technically-same-location shot can still fail COVERAGE by reading as a re-establish.

    STATE CONTINUITY, CARRYMARKS-SCOPED (rule 36): the join-check hard-gates STATE only on the beat's own declared `carryMarks` — present in both frames or absent in both, never present in one and gone in the other. Any OTHER visible prop or incidental set dressing (a bee casually holding pollen, say) is explicitly excluded from the hard gate and surfaces only as an advisory `flags` entry — logged for the record, never blocking.

    NO RE-MINT ON A CUT (rule 32): an `intentional_next_shot` next beat (the default) opens off the RAW harvested settle frame directly — no NB2 cleanup pass, no drift check, `cb_qa.check_remint` not in the path. Re-mint (and its drift check) is retained only for a `seamless_continuation` next beat, whose @图1 really is used as literal first-frame pixels — the cleanup pass earns its keep there. State continuity for a cut moves entirely to the post-fire join-check instead of a pre-fire drift check — verification happens on the actual rendered result, not a synthetic proxy for it.

    THE V4 TEMPLATE (rule 30 — `cb_segprompt.emit_v4`/`for_beat_v4`, the definitive builder for every beat, v3/v2/v1 all retired to loud-fallback-only): the six-line reference contract (Law 3 above, with the corrected @Video1 line); one continuous `prompt` field carrying the opener, character bindings, an OPEN-ON sentence (Layer 2.3, intentional-cut beats only) then a labelled timing clock whose closing segment is always "settle in character:" (Law 9); `acting_rules` its own top-level field (Layer 2.6); `continuity` naming the beat's own carry marks plus the show's warm light; `prohibited` merging the beat's own staging list with the six standing negatives, always at least six. Dialogue words never appear in the prompt (Law 6); V3 fires in as @Audio1 and drives generation, never stitched on after (rule 29 — no post voice swap, even in a future two-step fallback).

    THE GATES — MACHINE VS SHOWRUNNER: the machine checks what it CAN check, mechanically and vision-assisted — state continuity on declared carryMarks, spatial adjacency (Coverage), Clip QA (identity, anatomy, frozen, morph, pop-character), gag-element presence, the flag-only prompt-law lint (Laws 4/7/8, settle distinctiveness). Julian checks the only things that matter and that no gate owns: does it flow, is it funny, does the four-year-old watch it again. This is a RESERVED VERDICT, not a gap in the machine — no check is ever built to approximate it. The Layered Humour checklist rides above every mechanical gate and answers only to his eye: Fuzzby's chaos beats, Zenny's deadpan with its pause protected (Layer 2 lives in the pause; the settle is where her look breathes), the laugh-so-hard-they-miss-it ambition intact.

    THE WORKFLOW LOOP: fire → machine gates (two-tier join-check, Clip QA, prompt-law flags) → Julian's felt-intent verdict → sign → harvest → cut into the next beat, which declares its own junction type → assemble the next prompt from the skeleton plus its own creative slots → fire. Escorted through Scene 1 by hand; `cb_replicator.walk_scene` walks it thereafter under the identical laws — nothing beat-specific lives in its own code, only in the beat data. Gates per the escorted-run rules; scene boundaries reset to canon (the Scene Bubble Law).

    THE AUDIT MANDATE: a law living in a document can be forgotten; a law living in the emitter cannot. `PROMPT_LAWS_AUDIT.md` is a historical audit of v3's shape, not v4's plus this consolidated doctrine's — a fresh audit is a future ticket, not done automatically by this rule.

    THE ONE-RENDER ECONOMY (locked 2026-07-05, the full-reset night; wired into code the same night as the reset's first follow-up closer): one fire per artifact — the scene plate, a voice take, a keyframe, a beat render. Never a batch of exploratory seeds as the default. On a failed gate: ONE automatic re-fire, never blind. If that re-fire also fails: a HARD STOP with a diagnosis naming the layer at fault (keyframe / brief / reference / take, rule 3) — never a third roll. The voice pass is the same economy, stated explicitly: ONE directed V3 take per beat, never a batch of candidate reads — Julian's ear either approves it, or names the single correction for the one permitted re-fire; there is no "pick a favourite among several" step for voice, the same discipline the felt-intent gate already holds the beat render to. CODE-ENFORCED: `cb_beats.fire_next_beat`'s launch phase fires once, reads back the CLIP QA + JOIN CHECK verdicts `cb_beats.run` persists as `.qa.json`/`.join.json` sidecars, auto-retries once on a failure, and returns a `HARD_STOP` with a layer diagnosis (`cb_beats._layer_diagnosis`) if the retry also fails. `cb_replicator.walk_scene` carries the identical discipline for the scene's opening beat (`cb_beats._fire_gated`) and inherits it from `fire_next_beat` for every beat after. The old multi-seed default (`seeds=2`, "fire N takes, pick a winner") is retired everywhere it flowed — `fire_next_beat`, `walk_scene`, `cb_pipeline.relay_prepare`/`relay_approve`, and the Studio's seed-picker panel (`/api/beat-seeds`, app.html's "candidate seeds" UI) are all gone; a beat has exactly one official clip. Standard tier (`fast=False`) is the production default throughout.

    THE PRODUCTION LINE — THE CANONICAL STAGE MAP (locked 2026-07-05, the full-reset night; full prose version and rationale in `PRODUCTION_DOCTRINE.md`, mirrored in `REPLICATOR.md`): every stage names its gate holder. The machine checks; Julian signs. Nothing self-advances past any gate, ever (rule 1).
    - Stage 0 — Script-in: the script is the SOLE story source.
    - Stage 1 — Beat package → Gate 1: the script becomes the beat package (storyboard). Julian signs Gate 1.
    - Stage 2 — World and cast: the scene plate built then checked (`check_plate`) against the Crystal World Rule; character turnarounds verified against canon; the scene's ambient bed locked (the Scene Bubble Law). The machine checks; folds into Gate 2's review, no separate sign-off.
    - Stage 3 — Voice pass: one directed V3 take per beat, per the Character Voice Bible (the one-render economy, above). Julian's ear approves, or names the single correction.
    - Stage 4 — Gate 2: one anchor keyframe per SCENE (never per beat — a relay beat never gets its own), 2K, centre-safe, per-character action-state QA. Julian signs Gate 2.
    - Stage 5 — Gate 3: the escorted walk (`cb_replicator.walk_scene`) fires one render per beat, standard tier; machine gates (Clip QA, carryMarks-scoped state, Coverage, settle distinctiveness, the prompt-law lint); then the reserved felt-intent verdict, per beat.
    - Stage 6 — Gate 4: a retake fires only on Julian's own verdict, never a machine flag alone; one variable per re-fire (the retake protocol).
    - Stage 7 — Gate 5: post — settle-trim so the cut joins on living motion (the JOIN CONTRACT); beats assembled; the ambient bed continuous across the whole scene; music and grade; two masters delivered — the 16:9 feature master and a centre-safe 9:16 derivative. Julian's final-cut approval.
    A stage marked "the machine checks" is still a real gate — it just doesn't need Julian's own eyes on every single instance, the way Gate 1/2/3/4/5 do.

THE JUNCTION-TYPE PIVOT (Julian's ruling, 2026-07-05 — "the relay has been enforcing the wrong kind of join"): the whole six-attempt frame-one campaign (rules 15/19-30's history, through the locked v4 template) was solving the wrong problem — treating every relay beat as a SEAMLESS CONTINUATION of the previous instant, when most beats are actually a NEW GAG ARC: an editorial cut to a fresh camera setup, already at the new beat's own energy from its first frame. Every beat now declares its OWN junction type (a beat-level field, `junctionType`):
    - `intentional_next_shot` — THE DEFAULT for every beat unless the director's cut explicitly says otherwise. A new gag arc: a fresh camera setup, the beat's own energy already in frame from its first instant. @图1's job changes from "first frame, locked" to STATE REFERENCE ONLY: identity, temporary marks/wardrobe (a pollen moustache, dirt, a held object), lighting and world-position carry forward exactly; camera, pose, energy and motion do NOT. Worked template: "Fuzzby and Zenny appear exactly as in @图1 — carry the smeared moustache, the pollen, the lighting and their positions in the world — and open on a fresh camera setup for this beat's action."
    - `seamless_continuation` — ONLY where the director's own cut explicitly says the shot continues (an unbroken take spanning a beat boundary). Here, and only here, Layer 1's Law 5 (anti-hold) and Law 3's "@图1 the opening frame, locked" still apply in full: frame one IS @图1 exactly, then motion moves decisively away.
    A beat that does not declare a junction type is `intentional_next_shot` by default — never `seamless_continuation` by omission.
    THE JOIN-CHECK SPLITS TWO-TIER (supersedes rule 19's single check): STATE CONTINUITY — marks, wardrobe, lighting, geography — is the HARD GATE for every join, no exceptions, checked whether the beat is a cut or a continuation. FRAME IDENTITY (does beat N+1's opening frame literally match beat N's ending frame) is checked ONLY on a declared `seamless_continuation` join; an `intentional_next_shot` join is never held to frame-identity, since a cut is expected to look different.
    Harvest and re-mint (rules 21-22) are UNCHANGED by this pivot — the anchor still carries state forward exactly as before; only what the NEXT beat's prompt asks that state to do (be repeated vs. be referenced) and what the join-check demands of it (frame match vs. state match) changes.
    Rule 28's Layer 1 (laws 2, 3, 5) and its workflow loop are updated in the same commit to state the junction-type branch as the current constitution — this rule stands as the record of why.

RE-MINT SCOPING (Julian's ruling, 2026-07-05, the same session as the junction-type pivot): rule 22's "re-mint is standard for every link now, not QA-triggered" is SUPERSEDED — re-mint is conditional on the NEXT beat's own declared junction type, not automatic on every harvest. For an `intentional_next_shot` next beat (the DEFAULT, rule 31): the raw HARVESTED settle frame IS @图1 directly — no NB2 re-mint pass, `cb_qa.check_remint` not in the path at all. @图1 is a state reference for this junction type (identity, marks, lighting, position — never a pixel-perfect anchor), so the NB2 cleanup pass (removing compression artifacts and motion blur before something is presented as "frame one, exactly") buys nothing; skipping it saves a generation call, cost and latency for the common case. Harvest itself is UNCHANGED — the sharpest frame in the settle window is still selected exactly as before (rule 21).
    Re-mint is RETAINED, not deleted, for exactly two cases: (1) a next beat declared `seamless_continuation` — its @图1 really is used as literal first-frame pixels, so the cleanup pass still earns its keep; (2) any future image-to-video use where the frame is genuinely literal first-frame pixels (e.g. a true FLF2V call, or the second pass of a rule-29 two-step fallback) — no code path exercises this second case today; when one is built, it must call `cb_scene.remint_settle_frame` the same way the seamless path already does, never skip it.
    STATE CONTINUITY MOVES ENTIRELY TO THE POST-FIRE TWO-TIER JOIN-CHECK (rule 31's `check_join_state`, a hard gate on every join regardless of junction type) for an `intentional_next_shot` beat — there is no more pre-fire drift check to catch a state mismatch before the next beat launches; the join-check on the ACTUAL rendered clip is the only guard, and it runs after the fire, not before. This is a deliberate trade: one less synthetic image in the chain, verification happens on the real result instead of a proxy for it.
    Mechanically: `cb_scene.relay_source_for` now decides harvest-vs-remint by the RESOLVING beat's own junction type (never merely "does a re-mint file happen to exist on disk" — a stale re-mint from before this ruling, or from a differently-typed sibling beat, must never be silently preferred). `cb_beats.fire_next_beat`'s prepare phase decides by the NEXT beat's junction type and returns a new `anchor` key (always populated — the re-mint when one ran, the raw harvest otherwise) alongside the now-sometimes-`None` `remint`/`drift_check` keys; every caller (`cb_pipeline.relay_prepare`, `cb_replicator.walk_scene`, the Studio UI's relay banner) reads `anchor`, never assumes `remint` is the only shape a prepared anchor takes. The Studio UI's Approve-Anchor button previously hard-gated on `driftCheck.ok===true`, which would have left the button permanently disabled for every future intentional-cut beat now that `driftCheck` is legitimately `None`, not failing — fixed the same day this rule was written (`cb-studio/app.html`'s two relay-banner blocks).
    Rule 28's workflow loop is updated in the same commit to state re-mint as conditional, not unconditional.
